Especificação Técnica do Frontend (Next.js BFF) - Estúdio de Conteúdo IAEste documento define a arquitetura, o fluxo de dados e as diretrizes de interface de usuário (UI/UX) para o desenvolvimento do Estúdio de Conteúdo da Paulistana Empório. O frontend (Next.js) atuará como o Orquestrador Central, enquanto o n8n servirá como o Worker de produção.1. Visão Geral da ArquiteturaOrquestrador: Next.js (App Router) + Zustand (Gerenciamento de Estado).Worker de IA: n8n (Webhooks Síncronos).Fonte de Verdade: Supabase (PostgreSQL).Protocolo de Mídia (Strict): O Next.js NUNCA trafegará ou receberá arquivos binários (BLOBs) ou Base64 diretamente do n8n. Toda mídia será salva no Google Cloud Storage (GCS) pelo n8n. O n8n utilizará o nó Respond to Webhook para devolver exclusivamente URLs públicas ou JSONs de erro para o frontend.2. Estrutura do Estúdio (Abas e Presets)O dashboard será dividido em abas baseadas no formato de conteúdo: Vídeo, Carrossel e Blog. Cada aba possui configurações e System Messages específicos.2.1. Trava de System Messages (Security Layers)Os prompts base (System Messages) serão divididos no banco de dados e na UI (Store Zustand) em duas camadas para evitar falhas humanas:Locked / Essential Rules (Invisível/Bloqueado na UI): Regras de estrutura JSON, travas de HTML para blogs, limitações de caracteres de TTS, regras de layout do Satori, etc.Editable (Inputs para o Gestor): Persona do agente, tom de voz, nicho alvo, e Call to Action (CTA).3. Orquestração e Fila de Produção (A Máquina de Estados)A geração de mídia não ocorre em paralelo para evitar bloqueios de Rate Limit nas APIs de IA. O Next.js gerencia uma Fila de Execução Sequencial, baseada em requisições HTTP aguardadas (await fetch).3.1. Ciclo de Vida Inicial (Roteiro)Usuário preenche o briefing e clica em "Gerar Roteiro".Next.js cria um UUID4 único da campanha e insere um registro no Supabase.Next.js dispara um Webhook para o n8n e aguarda a resposta HTTP (que conterá o JSON do roteiro aprovado ou um erro).Em caso de sucesso, a UI atualiza renderizando as cenas na Linha do Tempo.3.2. A Fila Sequencial e Resiliência HTTP (Mídias)Quando o Roteiro é renderizado, o Next.js monta a fila de mídia e dispara cena a cena utilizando blocos try/catch rigorosos:Padrão de Requisição:O Frontend faz um POST para o Webhook de imagem/áudio da Cena 1.O n8n processa, gera o ativo, sobe no Bucket (GCS) e usa o nó Respond to Webhook para devolver { "success": true, "url": "https://..." }.O Frontend recebe o HTTP 200, atualiza o Supabase/Store, pinta a Cena 1 de "Verde" na UI, e emenda o fetch da Cena 2.Regra de Falha (Skip & Highlight):Se o n8n devolver um erro HTTP 500 ou a requisição der timeout (catch (error)), a fila NÃO DEVE PARAR.O Next.js marca o status do bloco atual como error, exibe uma borda vermelha na UI para chamar a atenção do usuário, e dispara automaticamente a próxima cena. O usuário tratará a falha visualmente depois.4. UI/UX: Linha do Tempo Horizontal (Timeline Editor)O acompanhamento da produção ocorrerá em uma interface de Linha do Tempo inspirada em softwares de edição de vídeo (Premiere/CapCut).Colunas: Cada cena do roteiro representa uma coluna.Trilhas (Tracks):Track Superior (Visual): Status de carregamento, Preview da Imagem (via tag <img> ou <video>) e dropdown de Estilo de Animação (zoom_in, pan_right).Track Inferior (Áudio): Status, Player de áudio <audio controls> e a narração textual (editável).Transparência de Prompts: O prompt de IA usado para gerar o visual ou o áudio deve estar visível num textarea expansível abaixo da mídia.Retry Cirúrgico: Cenas marcadas como error (por falha no HTTP) ou reprovadas visualmente pelo usuário terão um botão "🔄 Tentar Novamente". O usuário pode editar o prompt exibido na UI e clicar no botão para re-disparar apenas aquela tarefa HTTP específica.5. Workflows de Geração de Imagem (Bifurcação)Ao ler o Roteiro JSON gerado, o Next.js decidirá qual Webhook de imagem chamar no n8n.Fluxo A: Com Slugs (Produto Real)Gatilho: JSON da cena diz usa_referencia: true e contém slug_produto (ex: "maca_peruana").Ação: O Next.js busca a imagem limpa do produto no banco de dados, anexa a URL base na requisição e chama o Webhook de Image-to-Image no n8n, aguardando a devolução do link processado via Respond to Webhook.Fluxo B: Sem Slugs (B-Roll Genérico)Gatilho: JSON da cena diz usa_referencia: false.Ação: Next.js ignora a busca de produto e chama o Webhook de Text-to-Image puro no n8n.6. Lógica Específica: O "Human-in-the-Loop" dos CarrosséisA aba de Carrosséis não segue um fluxo contínuo e automático para evitar gasto de servidor com diagramações sobre imagens defeituosas.Passo 1 (Geração do Fundo): A fila sequencial pede ao n8n que gere apenas a imagem de fundo via IA. O n8n devolve a url_fundo na resposta HTTP.Pausa Estratégica: O Next.js recebe a url_fundo, exibe a imagem base para o Diretor de Arte (usuário) e exibe os inputs de texto, cores e tipografia ao lado.Passo 2 (Renderização API Satori): Se o usuário aprovar o fundo gerado (ou enviar sua própria foto de fundo localmente), ele clica no botão secundário "🎨 Aplicar Textos". O Next.js envia o JSON completo de diagramação para a API Satori e atualiza a cena com a url_final.7. Renderização Final (Aba Vídeo)7.1. Variáveis de Orquestração (Store)Além do script, o Zustand deve armazenar metadados do projeto na tela:Duração de cada áudio (ex: 4.5s) que servirá de limitador para a animação da imagem correspondente.Trilha Sonora (BGM): Dropdown com URLs do Supabase (ex: Lofi, Cinematic) ou input para link externo.Toggle: [x] Adicionar Legendas Dinâmicas.7.2. O Gatilho FinalApós todas as trilhas da "Timeline" ficarem verdes (url_final e audio_url resolvidos), a interface disponibiliza duas opções:Baixar Assets (ZIP): Comprime as URLs recebidas nos steps anteriores e baixa as mídias isoladas.Renderizar Vídeo Final: Envia um Big JSON Payload para o nó final do n8n / script de renderização, contendo as durações, links visuais, links de áudio, música de fundo e preferências de legenda. O frontend aguardará a resposta HTTP deste webhook com a URL final do vídeo MP4 montado.
