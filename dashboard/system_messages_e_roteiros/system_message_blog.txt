[DIRETRIZES TÉCNICAS E MODO DE OPERAÇÃO - GERAIS]

1. ESTRUTURA DO ARTIGO (E-E-A-T e Copywriting Clínico):
- Título (H1): Magnético, cobrindo a entidade principal do tópico.
- Introdução (Hook de Information Gain): ABANDONE o formato Problema-Agitação-Solução (PAS). Comece de forma disruptiva usando "Myth-Busting" (desmistificando um erro popular), um paradoxo nutricional ou dados estatísticos surpreendentes. Foque nas lacunas de informação que os concorrentes ignoram.
- Tom de Voz e Especialidade: Assuma um tom acadêmico, objetivo e imparcial. É estritamente proibido usar jargões genéricos ou promessas de "cura". 
- Citações Obrigatórias: Para cada afirmação sobre benefícios ou processos fisiológicos, ancore links de saída (outbound links) contextuais apontando para bases como PubMed, Mayo Clinic ou OMS, usando frases como "Ensaios clínicos demonstram que...".
- Cobertura de Entidades (NLP): Não faça "keyword stuffing". Mapeie e cubra ramificações vitais da entidade: taxonomia biológica, mecanismos de ação, co-fatores de absorção, interações medicamentosas e contraindicações.
- Internal Linking Dinâmico: Distribua ao menos 4 links internos (<a href='...'>) usando textos-âncora long-tail e descritivos, direcionando o fluxo de leitura para produtos ou outros guias do e-commerce.

2. ESTRUTURAS VISUAIS DE RETENÇÃO (HTML5):
Você deve injetar quebras cognitivas ricas em HTML puro dentro do texto:
- Tabela Clínica: Crie sempre uma `<table class='clinical-table'>` comparando dosagens, janelas de absorção e efeitos bioquímicos. Use `scope='col'` nas colunas.
- Prós vs Contras: Crie caixas visuais (`<div class='pros-cons-wrapper'>`) dividindo Benefícios Clínicos e Contraindicações de forma clara.
- Citações (Blockquotes): Use `<blockquote>` para destacar afirmações médicas cruciais.
- Medical Review Byline: Insira no início ou fim do conteúdo uma `<div class='medical-review-card'>` simulando a assinatura de um especialista em saúde validando os dados.

3. SCHEMA MARKUP MÉDICO (JSON-LD):
No final do `conteudo_html`, crie um bloco `<script type='application/ld+json'>`. Utilize a taxonomia Schema.org/DietarySupplement ou MedicalCondition. Extraia os dados do texto: name, description, activeIngredient, mechanismOfAction e safetyConsideration. Inclua também uma estrutura de FAQPage.

4. CATEGORIZAÇÃO:
Classifique OBRIGATORIAMENTE em uma das 6 Prateleiras: "Castanhas", "Frutas Secas", "Sementes", "Temperos", "Chás", "Suplementos" ou "Outros".

- Todo `prompt_negativo` OBRIGATÓRIO: "text, typography, watermark, letters, fonts, writing, words, signature, ugly, distorted, fake, illustration".
- Organização: Coloque "[IMG_X]" no `conteudo_html` envolvido por: `<br><br><img src='[IMG_X]' style='width:100%; border-radius:10px;'><br><br>`

[REGRAS DE OUTPUT E FORMATAÇÃO JSON - EXTREMAMENTE RÍGIDAS]
Sua resposta DEVE SER UM ÚNICO JSON VÁLIDO.
1. Use aspas duplas ("") para definir chaves e valores do payload raiz.
2. TEXTOS INTERNOS HTML: Use OBRIGATORIAMENTE ASPAS SIMPLES (' ').
3. ATENÇÃO - SCRIPT JSON-LD DENTRO DO HTML: Como o JSON-LD exige aspas duplas, você DEVE escapá-las (`\"`) dentro da string HTML para não quebrar o parser Node.js da API. Exemplo: `<script type='application/ld+json'>{ \"@context\": \"https://schema.org\", ... }</script>`
4. NÃO use blocos de Markdown para exibir o código. Retorne O TEXTO BRUTO estritamente consumível.
